# 🏡 Housing Price Predictor

Welcome to the **Housing Price Predictor**! This is a complete, beginner-friendly Machine Learning project designed to teach the core concepts of **Multiple Linear Regression**. 

Instead of treating the AI like a "magic black box," this project explicitly shows how the model uses mathematical weights and relationships across different features to predict real-world house prices.

---

## 🚀 Quick Start Guide

### 1. Requirements

Ensure you have Python installed on your system. Then, install the required dependencies (like `pandas` and `scikit-learn`):

```bash
pip install -r requirements.txt
```

### 2. Running the Application

This software is controlled via a graphical Command-Line Interface (CLI). To launch it, run:

```bash
python main.py
```

You will see an interactive menu:
1. **Train the Machine Learning Model:** Reads data, calculates relationships, and saves the trained brain as `model.pkl`. *(Must be run at least once)*
2. **Predict a House Price:** Manually enter Area, Bedrooms, and Age to get a price prediction.
3. **Generate Dataset Visualizations:** Draws a neat graph visualizing how area scales with price.
4. **Exit**

---

## 🧠 How it Works: The Machine Learning Concept

This project uses **Multiple Linear Regression**. 
When you choose to train the model, the algorithm looks at our dataset (`data.csv`) and tries to figure out a mathematical formula that connects our features (Area, Bedrooms, Age) to the target value (Price).

The equation it attempts to build looks like this:
```
Price = (Weight_1 × Area) + (Weight_2 × Bedrooms) + (Weight_3 × Age) + Base_Price
```

Once the model learns the optimal "weights" (coefficients) and the Base Price (intercept) that result in the least amount of errors, it saves those numbers into a file called `model.pkl`. 

Later, when you input brand new house details, it simply plugs your answers into that learned formula to calculate the final price!

## 📂 Project Structure

This codebase is split into modular, easily digestible scripts so you can isolate and study the different parts of a standard ML pipeline:

* **`data.csv`**
  The sample dataset acting as historical records. It contains 20 examples of houses with varying combinations of square footage, bedrooms, and age, alongside their actual market price.
  
* **`train.py`**
  The brain of the operation. This script:
  1. Loads the CSV using `pandas`.
  2. Splits the data (80% used to teach the model, 20% held back to test its accuracy).
  3. Uses `LinearRegression` from `scikit-learn` to learn the weights.
  4. Evaluates itself using Mean Squared Error (MSE).
  5. Exports the trained model via `joblib`.

* **`predict.py`**
  The user-interaction layer. This script catches the `model.pkl` generated by `train.py`, sets up user input prompts in the terminal, structures your answers into a DataFrame, and triggers the `.predict()` function to give you an estimated house price.

* **`main.py`**
  The controller. This runs an infinite continuous loop, giving you a clean menu to trigger training, prediction, or visual components effortlessly. 

* **`requirements.txt`**
  The list of required third-party Python modules.

## 🧰 Tech Stack

* **Python 3:** Core programming language.
* **Pandas:** For reading the CSV files, previewing data, and structuring input matrices.
* **Scikit-Learn (sklearn):** The premiere Machine Learning library, providing the Linear Regression model, training algorithms, and testing mechanics.
* **Matplotlib:** Used for generating beautiful visual graphs.
* **Joblib:** Required to save (serialize) our trained model array into a file so it doesn't need retraining per interaction.
